# NICE 본인인증

## 개요

- 운영 기관: NICE 평가정보
- 공식 사이트: https://www.niceapi.co.kr
- 우리 사용: 가입자 본인확인 (이름·휴대폰·DI/CI 발급)

## 도입 시점

**Phase 3 (사용자 출시 직전, sub-project 3 인증 단계)**.
Phase 1-2엔 자리만 (`crates/data-clients/nice/` 빈 README).

## 인증 / 등록

- CP(Content Provider) 등록 신청 → 검토 1-2주
- 인증서 + API 키 발급
- 테스트 환경 → 실서비스 환경 전환

## 비용

- **건당 100-300원** (계약 조건별)
- 우리 추정: 사용자 1만 가입 = 월 100-300만원
- 대안 검토: KCB, Toss 본인인증 (가격 비교 필요 — Phase 3에서)

## 핵심 흐름 (OIDC 또는 self-API)

1. 사용자가 "본인인증" 버튼 클릭
2. NICE 표준창 열림 (휴대폰 / 카드 / 공인인증서)
3. 인증 성공 → 우리 콜백으로 토큰 전달
4. 백엔드가 토큰으로 사용자 정보 (이름·DI/CI) 조회
5. DI/CI를 사용자 레코드에 저장 (해시 또는 암호화)

## 환경변수

```
NICE_API_KEY=
NICE_CLIENT_ID=
NICE_CERT_PATH=  # 인증서 경로 (Vault에 저장)
```

## 보안 정책

- ❌ DI/CI 평문 저장 금지 → AES-256-GCM 암호화 또는 SHA-256 해시
- ❌ 응답 raw 그대로 로그에 남기기 금지 (PII 마스킹)
- ✅ Audit log에 인증 시도/성공/실패 모두 기록 (DI는 마스킹)
- ✅ 5분 이내 응답 만료 (replay 방지)

## 한국법 준수 (PIPA)

- 본인확인은 *최소 정보*만 (이름 + DI/CI)
- 휴대폰 번호 등 추가는 별도 동의
- 보관 기간 명시 (5년 또는 가입 유지 기간)
- 탈퇴 시 즉시 삭제 (또는 가명화)

## Circuit Breaker 정책

- timeout: 30초 (사용자가 입력하는 동안)
- retry: 0회 (사용자가 다시 시도)
- fallback: 명확한 에러 메시지 ("본인인증 서비스가 응답하지 않아요")

## 대안 비교 (Phase 3 결정)

| 제공자 | 장점 | 단점 |
|--------|------|------|
| **NICE** | 가장 보편 | 가격 |
| KCB | 비슷 | 사례 적음 |
| Toss 본인인증 | 토스 사용자 친숙 | 가격·사례 |
| 카카오 인증 | 무료 (소셜 로그인 일부) | DI/CI 미발급 (별도) |

→ Phase 3에서 ADR로 정식 결정.
