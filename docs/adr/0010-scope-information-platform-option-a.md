# ADR-0010: 범위 — 산업용 부동산 정보 플랫폼 (옵션 A)

| | |
|---|---|
| 작성일 | 2026-05-01 |
| 상태 | Accepted |
| 결정자 | 운영자 |

## 컨텍스트

브레인스토밍 결과 프로젝트는 *산업용 부동산 정보 플랫폼* — 매물 검색·등록·연락처 노출 + 공공 데이터 자동 보강 + 분석 도구 + 제조업체 디렉토리. AI가 *사용자 노출 텍스트를 생성*하는 기능은 미포함 (옵션 A). LLM 활용은 검색 의미 매칭(임베딩, ADR-0011)에 한정.

## 결정

- **범위**: 옵션 A — 데이터 플랫폼 (정보 + 연락처)
- **AI 텍스트 생성**: 사용자 노출 0
- **임베딩**: 시맨틱 검색·유사 매물 추천 (Phase 3+, ADR-0011)
- **메시징**: Phase 1엔 연락처 노출만, 향후 인앱 메시징 (Phase 2+)
- **결제·전자계약·에스크로**: 범위 외 (직접 거래 중개 안 함)
- **AI 어시스턴트 챗 (옵션 C)**: 별도 모듈로 미래 추가 가능 — `apps/ai-assistant/` 자리만 비워둠

## 대안

- **옵션 B (AI 보고서 자동 생성)**: 환각 위험 + verify_citations 의무 + 책임 부담. 기각
- **옵션 C (AI 챗)**: SSS 데이터 플랫폼과 분리해서 *별도 앱*으로만 가능. 메인 시스템 범위 외
- **옵션 D (전자 계약 + 결제 풀 중개)**: 자본시장법·전자상거래법·부동산거래신고법 부담. Phase 4+ 별도 결정

## 결과

- 긍정: 환각 위험 0, 매물 데이터의 정확성 = 기관 + 사용자 책임, 컴플라이언스 부담 낮음, 메인 시스템 단순함, AI 모듈은 *분리* 가능 (격리)
- 부정: 직접 매매 중개 시장 진출 시 별도 결정 필요, AI 자동 보고서 같은 차별점 빠짐 (대신 *공공 데이터 통합*이 차별점)
- 영향 영역: 모든 도메인 (생성 코드 0), 인증 (NICE 본인인증은 매수 의도자 신뢰용), 결제 (광고+구독만)

## 재검토 트리거

- 사용자가 "이 매물 분석해줘" 자연어 인터페이스 강력 요구 시 → 옵션 C `apps/ai-assistant/` 활성화
- 직접 거래 중개 시장 진출 결정 시 (자본시장법 검토 + 별도 ADR)
- AI 자동 분석 보고서가 경쟁사 차별점이 될 때
