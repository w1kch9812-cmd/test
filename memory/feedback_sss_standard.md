사용자는 프로젝트 품질을 "하이엔드 엔터프라이즈 프로덕션 **SSS급**"으로 요구.

"A급"과 "SSS급"의 차이를 스스로 명확히 제시:
- 단순 파일 계층 + SSOT 원칙은 **A급**
- 폴더 단위 SSOT + 500줄 규칙 + 하네스 엔지니어링 + 감사 가능성 = **SSS급**

**Why**: 이전 SSMS 프로젝트의 1,349줄/1,447줄 거대 파일을 "SSS 아닌 A급"으로 격하. 즉 사용자는 *이름만 SSOT인 거대 파일*을 배격하고 **세밀한 관심사 분리 + 에이전트 컨텍스트 친화성**을 핵심으로 봄.

**How to apply**:
- 500줄 상한 엄수
- 한 폴더 = 한 도메인
- `.claude/hooks/` 로 정책 자동 강제 (구현 예정)
- 읽기 전용 감시자 에이전트 도입 (docs-auditor, compliance-reviewer 등)
- "아첨하지 말고 솔직한 평가" — 사용자가 직접 "SSS급이 맞느냐"고 반문했을 때 과장 없이 부족한 점 지적 요구
- 하네스 엔지니어링(hooks/agents/skills/MCP/permissions/keybindings) 의식적으로 설계

구체적 하네스 레이어는 프로젝트 진행하며 추가 예정.
