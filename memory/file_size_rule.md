type: feedback

모든 문서와 코드 파일은 **500줄 이하**를 목표. 1,500줄은 이미 안티패턴.
초과할 것 같으면 **폴더로 분해** (예: schema.md → schema/auth.md + schema/site.md + ...).

**Why**: 사용자(2026-04-22)가 이전 프로젝트(SSMS)의 docs/schema.md 1,349줄, site-builder.md 1,447줄을 비판. "SSOT를 파일 단위로 하면 한 파일이 거대해지는데 그건 구조적이지 않다 — 폴더 단위 SSOT가 진짜 구조적"이라는 피드백. 에이전트 컨텍스트 경제성과 병렬 편집을 위한 실용 기준.

**How to apply**:
- 새 문서 작성 시 500줄 넘을 것 같으면 처음부터 폴더로 시작
- 기존 문서가 커지면 즉시 분해, 리팩토링 미루지 말 것
- 각 폴더에 README.md 를 목차로 두고, 소비자는 README부터 읽음
- 코드 파일도 동일 — 함수/타입 단위로 분해
- 규칙 위반 발견 시 docs-auditor 에이전트가 경고 (구현 예정)
