2026-04-22 사용자가 프로젝트 범위를 **옵션 A: 데이터 플랫폼**으로 확정.

## 옵션 A의 의미

- 공공데이터를 가져와서 **지도·표·차트·검색 UI로 보여주는 사이트**
- 사용자에게 노출되는 텍스트는 데이터 그 자체 또는 사람이 작성한 것
- LLM·AI·MCP가 사용자 응답을 생성하지 않음
- 환각 위험 0, 환각 검증 불필요

## 향후 옵션 C (AI 챗) 추가 시 규칙

- **별도 앱**으로 분리: `apps/ai-assistant/`
- **별도 도메인**: `ai.{메인도메인}` 권장
- **별도 빌드·배포 파이프라인**
- 메인 앱(`apps/web`)에서 AI 모듈 import 금지
- 그때 MCP, LLM 클라이언트, verify_citations 한꺼번에 도입

## MCP 3종의 현재 위치

- 개발자가 Claude Code 세션에서 도메인 탐색용으로만 사용
- 코드(apps/web)에 import 금지
- 도메인 지식 학습용 reference 자료로는 활용 가능 (MIT)

**Why**: 옵션 B/C 가정으로 복잡도를 끼워 넣지 않기 위함. SSS급의 핵심 = 불필요한 복잡도 제거.
**How to apply**: 메인 시스템 설계·구현 시 LLM/AI 라이브러리 의존성 추가 거부. 사용자가 명시적으로 "AI 챗 만들자"라고 할 때까지 별도 모듈 자리만 비워둠.
